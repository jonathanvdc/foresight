trait Command[NodeT]

A Command encapsulates a single, replayable edit to an e-graph.

Commands are pure values that describe what to do; they donâ€™t perform any mutation until applied. This design allows callers to build, simplify, batch, or reorder edits before committing them to an EGraph.

Parallelization strategy to label and distribute any internal work.

Mapping from virtual symbols to concrete calls available before this command runs. This is used to ground virtual references present in uses.

  • maybeNewGraph is Some(newGraph) if any change occurred, or None for a no-op.

def definitions: Seq[Virtual]

The virtual e-class symbols this command promises to define when executed.

The virtual e-class symbols this command promises to define when executed.

Each virtual symbol represents an e-class that will be materialized in the target e-graph and receive a concrete EClassCall. After a successful apply, the returned reification map contains an entry for every symbol listed here.

def simplify(egraph: EGraph[NodeT], partialReification: Map[Virtual, EClassCall]): (Command[NodeT], Map[Virtual, EClassCall])

Returns a semantically equivalent command that is cheaper to execute on the given e-graph.

Returns a semantically equivalent command that is cheaper to execute on the given e-graph.

Typical simplifications include:

  • eliminating unions whose endpoints are already congruent
  • dropping inserts of nodes already present
  • narrowing work by pre-binding outputs in the returned partial reification

The returned partial map contains bindings this command can prove without running, which callers may compose across multiple commands to reduce future work.

Value parameters

Target e-graph used as context for optimization.

Known virtual-to-concrete bindings available upstream.
